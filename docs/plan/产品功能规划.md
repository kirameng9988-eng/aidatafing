# 需求工单管理系统 - H5移动端产品功能规划 v2.0

## 一、产品概述

### 1.1 产品定位
需求工单管理系统H5移动端，聚焦于**AI数据寻源**和**工单管理**两大核心功能，为用户提供智能化的数据资源查找和工单协作能力。

### 1.2 目标用户
- 数据分析师：快速找到所需数据资源
- 产品经理：提交数据需求和工单
- 开发人员：处理工单和提供数据支持
- 管理者：监督工单处理效率

### 1.3 核心价值
- 🤖 **AI智能寻源**：通过自然语言对话快速找到数据资源
- 📱 **移动办公**：随时随地处理工单
- 📊 **可视化进度**：清晰了解工单处理进度
- ⚡ **高效协作**：快速响应和处理

## 三、详细功能设计

### 3.1 AI数据寻源助手（首页）

#### 功能描述
通过AI对话的方式，帮助用户快速找到所需的数据资源，是系统的核心功能页面。

#### 页面元素
1. **顶部导航栏**
   - 系统标题"AI数据寻源助手"
   - 历史记录按钮（右侧）

2. **对话区域**
   - 欢迎消息和快捷问题
   - 用户消息（右侧蓝色气泡）
   - AI回复（左侧白色气泡）
   - 资源推荐卡片

3. **输入区域**
   - 文本输入框（自适应高度）
   - 发送按钮（圆形）

4. **历史记录侧边栏**
   - 从右侧滑出
   - 显示所有历史对话
   - 点击可加载历史对话

#### 资源推荐卡片内容
- 资源名称
- 数据库.表名
- 数据类型（MySQL/Hive等）
- 资源描述
- 主要字段
- 更新频率
- 数据质量
- 负责团队

#### 交互说明
- 输入问题 → 显示打字动画 → AI智能匹配资源 → 展示资源卡片
- 点击快捷问题 → 自动填充并发送
- 点击历史按钮 → 打开历史记录侧边栏
- 点击历史记录项 → 加载该对话
- 所有对话自动保存到LocalStorage

#### 智能匹配逻辑
```
关键词匹配规则：
- "用户" / "行为" / "画像" → 用户相关数据表
- "订单" / "交易" / "购买" → 订单相关数据表
- "销售" / "客户" / "业绩" → 销售相关数据表
- "财务" / "发票" / "流水" → 财务相关数据表
```

### 3.2 工单管理

#### 功能描述
展示所有工单，支持筛选、搜索和新增工单。

#### 页面元素
1. **顶部导航栏**
   - 页面标题"工单管理"
   - 新增工单按钮（+）

2. **搜索栏**
   - 搜索输入框
   - 实时搜索

3. **状态筛选Tab**
   - 全部
   - 待处理
   - 进行中
   - 已完成

4. **工单卡片列表**
   - 工单标题
   - 工单编号
   - 优先级标签（高/中/低）
   - 状态标签
   - 工单类型
   - 更新时间
   - 进度条（如有进度）

5. **新增工单弹窗**
   - 工单标题（必填）
   - 工单描述
   - 优先级选择（高/中/低）
   - 工单类型选择
   - 提交按钮

#### 交互说明
- 点击新增按钮 → 弹出创建工单表单
- 填写表单 → 提交 → 显示成功提示 → 刷新列表
- 输入搜索内容 → 实时过滤工单
- 点击Tab → 切换不同状态的工单
- 点击工单卡片 → 跳转到工单详情

---

### 3.3 工单详情

#### 功能描述
展示工单的详细信息和处理进度。

#### 页面元素
1. **顶部导航栏**
   - 返回按钮
   - 工单编号
   - 更多操作按钮

2. **状态和进度卡片**
   - 当前状态标签
   - 进度百分比
   - 进度条

3. **基本信息区**
   - 工单标题
   - 工单描述
   - 优先级
   - 工单类型
   - 创建时间
   - 创建人
   - 负责人

4. **进度时间轴**
   - 时间节点列表
   - 每个节点显示：时间、操作人、操作内容、状态图标

5. **操作区**
   - 处理工单按钮
   - 添加评论按钮
   - 关闭工单按钮

#### 交互说明
- 点击返回 → 返回工单列表
- 点击处理工单 → 弹出处理表单（进度滑块+说明）
- 点击添加评论 → 弹出评论输入框
- 点击关闭工单 → 确认后关闭工单

## 四、技术方案

### 4.1 技术栈
- **前端框架**：纯HTML + Tailwind CSS
- **UI组件**：自定义组件 + FontAwesome图标
- **交互逻辑**：原生JavaScript
- **数据存储**：LocalStorage（历史记录）
- **页面路由**：通过页面跳转模拟

### 4.2 设计规范

#### 颜色规范
- 主色：#3B82F6 (蓝色)
- 成功色：#10B981 (绿色)
- 警告色：#F59E0B (橙色)
- 危险色：#EF4444 (红色)
- 中性色：#6B7280 (灰色)

#### 字体规范
- 标题：16-20px，粗体
- 正文：14-16px，常规
- 辅助文字：12px，常规

#### 间距规范
- 页面边距：16px
- 卡片间距：12px
- 元素间距：8px

## 六、AI数据寻源功能详解

### 6.1 模拟数据资源库
```javascript
const dataResources = {
    user: [用户基础信息表, 用户行为日志表, 用户画像标签表],
    order: [订单主表, 订单明细表],
    sales: [销售业绩表, 客户信息表],
    finance: [财务流水表, 发票信息表]
}
```

### 6.2 对话流程
1. 用户输入问题
2. 系统识别关键词
3. 匹配对应的数据资源
4. 生成资源推荐卡片
5. 保存到历史记录

### 6.3 历史记录功能
- 自动保存所有对话
- 最多保存50条记录
- 存储在LocalStorage
- 点击可重新加载对话

### 6.4 快捷问题预设
- "我想找用户行为数据"
- "需要销售相关的数据表"
- "有哪些财务数据可用？"
